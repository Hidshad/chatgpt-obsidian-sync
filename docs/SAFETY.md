# Safety

## 设计原则

`v0.1.0-preview` 只聚焦一个主流程：把当前打开的 ChatGPT 会话完整导出到本地 Obsidian，并在同一个会话页继续进行文字实时同步。

安全策略优先保证：

- 只在用户明确点击后开始同步。
- 非普通会话页静默待机。
- 预览版建议一次只同步一个 ChatGPT 会话标签页。
- 不提供一键导出全部历史会话。
- 不保存 token、cookie 或 Authorization header。
- 对话内容只发送到 `http://127.0.0.1:8765`。

## Preview 行为

- 点击“开始同步当前会话”时，如果本地没有完整基线，会先后台校准一次。
- 完整基线成功后，才开启当前会话文字实时同步。
- 如果完整导出失败，不会开启实时同步。
- 重新生成当前会话笔记只根据本地 SQLite 数据重建 Markdown，不访问 ChatGPT。
- popup 关闭后，当前同步或完整校准由 background 继续持有状态。

## 不保存 token 的原则

项目不要求用户复制 Bearer token，不读取或保存 Authorization header，也不把 ChatGPT cookie 写入本地文件、数据库或配置。

## 推荐使用方式

- 手动同步当前重要会话。
- 每次同步后检查 Obsidian 输出。
- 使用合理的实时同步间隔。
- 如果 ChatGPT 出现验证、限制或异常，停止使用并稍后再试。

## 不推荐使用方式

- 多个 ChatGPT 标签页同时同步。
- 高频切换会话后立即同步。
- 自动遍历历史会话。
- 保存或复制 Bearer token。
- 抓取他人内容、共享账号内容或绕过访问控制。

## 与 OpenAI 的关系

本工具不是 ChatGPT 官方工具，与 OpenAI 无官方关联。用户需要自行确认自己的使用方式符合相关服务条款、组织政策和本地法律要求。
